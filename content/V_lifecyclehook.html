Vue 3 Composition API におけるライフサイクル

1. ライフサイクルとは

Vueコンポーネントが「生成→表示→更新→破棄」する一連の流れの中で、特定のタイミングで処理を実行できる仕組みです。 Composition APIでは以下のような関数で扱います： 

onMounted(() => { ... })      // マウント直後
onBeforeUnmount(() => { ... }) // 破棄前

2. 主なフック関数

タイミングフック名
マウント前onBeforeMount
マウント直後onMounted
更新前onBeforeUpdate
更新後onUpdated
破棄前onBeforeUnmount
破棄後onUnmounted

3. ライフサイクルが必要な理由

  • DOM操作や外部ライブラリ初期化は表示後（onMounted）でないと正しく行えない
  • イベントリスナ・タイマーの解除は破棄時（onUnmounted）に必要
  • コンポーネント単位で管理できるため、再利用性と保守性が向上

4. コンポーネントとの関係

VueはUIを小さなコンポーネント単位で構築します。
各コンポーネントが独立してライフサイクルを持つことで、必要なタイミングで適切な処理を行うことができます。

5. ページサイクルとの違い

比較項目ライフサイクルページサイクル
対象Vueの各コンポーネントブラウザ全体のページ（DOMContentLoadedなど）
実行範囲特定のVueコンポーネント内のみページ全体に一度だけ
主な目的UI制御・状態管理ページ初期化やアンロード処理など

まとめ

Vueのライフサイクルは、コンポーネントの状態管理とタイミング制御に不可欠であり、ページ全体ではなく“部品”単位で完結する仕組みです。
