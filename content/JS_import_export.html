JavaScriptのmoduleにおける import と export

概要

  • export : モジュールから関数・変数・クラスなどを外部に公開する
  • import : 他のモジュールから公開された要素を取り込む

用途

  • コードの分割と整理
  • 再利用性の向上
  • 依存関係の明示

サンプル

1. ファイル構成

project/
├─ math.js
└─ main.js

2. math.js : exportの使い方

// 名前付きエクスポート(複数可)
    export function add(x, y) {
      return x + y;
    }

    export const PI = 3.14;

    // デフォルトエクスポート(1つだけ)
    export default function multiply(x, y) {
      return x * y;
    }

3. main.js : importの使い方

// 名前付きimport(名前一致が必要)
import { add, PI } from './math.js';

// デフォルトimport(名前は自由に設定可)
import multiply from './math.js';

console.log(add(2, 3));      // 5
console.log(PI);             // 3.14
console.log(multiply(4, 5)); // 20

ポイント

種類 特徴 書き方
名前付きエクスポート 複数可・{}で指定してimport export { name } / import { name }
デフォルトエクスポート 1つだけ・名前は自由 export default ... / import 任意名

Vue.jsなどのフレームワークでも頻繁に使われます．
